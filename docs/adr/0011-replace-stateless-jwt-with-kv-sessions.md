# Replace Stateless JWTs with KV Sessions for Multi-User Auth

Multi-user support requires per-user identity, per-device sessions, and server-side revocation — none of which the legacy role-only JWT (signed with the shared admin key) could express. Login state is now a server-side Session: an opaque cookie token backed by a KV record (Cloudflare KV with native TTL in production, the local file store in standalone mode), with PBKDF2-hashed passwords under a per-user salt (WebCrypto-native, no new dependency), per-user API Tokens replacing the single global bearer token, and password changes revoking the user's other sessions. We considered a D1/SQLite session table (strong consistency, one drizzle code path) and keeping stateless JWTs (smallest change, no revocation), and accepted Cloudflare KV's eventual-consistency window — visible at worst as a rare re-prompt just after login — because sessions then live beside the existing global config with no new database dependency.
