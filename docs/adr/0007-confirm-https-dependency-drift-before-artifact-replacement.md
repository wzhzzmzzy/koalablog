# Confirm HTTPS dependency drift before Artifact replacement

Koalablog permits bounded absolute HTTPS imports but does not assume that a URL identifies immutable module bytes. Every Render Artifact therefore records a canonical manifest of each fetched module's final URL, UTF-8 byte count, and SHA-256, and the server includes that manifest in `artifactHash` while keeping network bytes out of `sourceHash`. If a rebuild for the same Source Hash produces a different dependency manifest, ordinary attachment returns `409 dependency_changed` and preserves the existing Current Artifact until an authenticated site owner explicitly confirms replacement with the exact current/proposed Artifact Hash pair; the server repeats validation and rejects stale confirmation if either side changed. Batch rebuilds report drift and never confirm it automatically. This trades automatic rebuild convenience for visible supply-chain changes without making Source Save or Source Hash calculation depend on network access or allowing a prior approval to authorize different bytes.
