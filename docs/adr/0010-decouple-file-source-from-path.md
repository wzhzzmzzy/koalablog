# Decouple File Source from File Path

A File's Source (Post or Memo) was previously recomputed from its path prefix on every save, so list membership could never differ from the path namespace, and retiring the Page and Wiki sources would have forced URL moves. Source is now persisted metadata assigned from the Path Prefix at creation and never re-derived on save; existing Page and Wiki files migrate to Memo with every URL unchanged. We chose this over physically moving files under `/memo/` behind a redirect table, because preserving every existing URL outweighs the oddity of files living outside `/memo/` appearing in the memos list. The `/posts` and `/memos` list pages are removed (the catch-all route resolves those paths as ordinary files), the index page switches between post and memo lists through query parameters, and the three article prefix routes collapse into the catch-all.
