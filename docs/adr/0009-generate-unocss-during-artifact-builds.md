# Generate UnoCSS during Artifact builds

Database-backed Svelte Source is unavailable to Astro's deployment-time UnoCSS scan, while loading an UnoCSS runtime on public pages would leave the SEO Snapshot unstyled without JavaScript and add per-visitor work. Koalablog therefore runs a pinned browser-safe UnoCSS `65.4.3` profile inside the lazy Artifact Worker, using `presetUno({ preflight: 'on-demand' })`, the shared font theme and shortcuts, Svelte-AST extraction limited to static class attributes and class directives, and explicit style directives. Generated utility selectors and any on-demand variable initialization are rooted under `:where([data-koala-artifact-root])`, then placed before Svelte's component CSS and stored in the ordinary Artifact CSS payload; the Preview iframe and public Page use the same root marker. Artifacts record the exact UnoCSS version and canonical configuration hash, and public pages load only the resulting checked stylesheet. Icons, broad script/text scanning, and dynamically constructed class fragments are excluded from v1 to keep extraction deterministic and avoid accidental Page Shell styling. Explicit Svelte global CSS and trusted runtime DOM changes remain allowed because the component is already trusted same-origin application code, but an unanchored global selector produces a visible non-blocking diagnostic rather than silently crossing the ordinary body-style boundary.
