# Store derived render artifacts for Svelte files

Svelte files remain ordinary server-persisted source files even when they are incomplete, invalid, or lack a usable render artifact. A successful client-side render build stores a size-bounded, versioned, self-contained JavaScript bundle, component CSS, and an SEO snapshot derived from the same source revision, avoiding per-visitor compilation and all server-side compilation or user-code execution. Saving or importing source never depends on compilation success; imports build artifacts afterward on a best-effort basis. The source row stores the canonical hash of Renderer Mode and Source content and updates it atomically with Source Save, so an older artifact becomes non-current immediately. A Current Artifact requires Svelte mode and a matching hash; a missing or mismatched artifact produces an explicit render failure instead of source exposure, stale output, or a draft/publication state. Existing artifacts keep their pinned Svelte version until rebuilt. Artifact requests always recheck current File access and lifecycle state: public responses revalidate rather than use immutable caching, private responses are not stored, and trashed or purged Files are unavailable.
