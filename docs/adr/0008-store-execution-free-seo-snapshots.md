# Store script-free SEO Snapshots

Koalablog's live Svelte component remains trusted same-origin application code, but its saved SEO Snapshot must not execute script before or in place of that component. The client therefore serializes the captured body through a deterministic filter that removes executable elements and URL schemes, event attributes, embedded browsing contexts, and extra style/head resources while preserving ordinary body structure, text, images, safe links, native form controls, and safe form submission; the server independently validates the submitted fragment and returns `422 invalid_snapshot` if executable code or a non-canonical representation remains. Form elements are not unwrapped or disabled, and safe action/formaction navigation remains available when JavaScript is absent or live mount fails. This deliberately trades exact fallback fidelity for predictable indexing and no-script use, and prevents HTML created through `{@html}` from running script early or twice without claiming that the later live mount is sandboxed or side-effect-free.
