# Trust Svelte files as application code

Svelte files may execute arbitrary Svelte code directly in the same-origin page because they are treated as site-operator-owned application code, not untrusted user content. The platform gives each component an explicit mount target limited to the blog page's File-body region, while the surrounding Page Shell remains platform-owned. We reject iframe, origin, and Shadow DOM isolation for public rendering, so this ownership boundary is an Interface contract rather than a security boundary: arbitrary trusted code can still reach the same-origin document. Anyone able to create or synchronise a Svelte file therefore has the same effective browser-side authority as deployed site code.
